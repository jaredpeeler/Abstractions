    RequireAuthentication

    If this is set to true then the custom API can only be called from within DirectScale secure infastructure such as CloudSpark, Retail Shop, or our azure API gateway.

    Declaration
    public bool RequireAuthentication { get; set; }
    Property Value
    Type Description
    System.Boolean
    Remarks

    Warning! If this is set to false, your API will be publically exposed and can be called by anyone! If false, authentication must be done manually. For example, checking for a secret key sent securely server by another server. See Headers.

    Note: Our authentication procedures do NOT verify authorization! If your custom API is doing something for a specific associate, make sure that the request was only initiated by that associate. See User.

    Route

    This is the route that will trigger the custom API to be called.

    Declaration
    public string Route { get; set; }
    Property Value
    Type Description
    System.String
    Remarks

    To call this custom endpoint the url path will be prefixed with "/Command/ClientAPI/". Do not set Route to a value starting with "/".

    Examples

    Setting Route to "AwesomeCompany/AwesomeEndpoint1" can be called from your corp admin domain + "/Command/ClientAPI/AwesomeCompany/AwesomeEndpoint1.
